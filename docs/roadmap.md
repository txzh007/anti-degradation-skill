- refine the main skill wording
- add more regression cases from real usage
- formalize a reusable test case template

- split the skill into composable modules
- add specialized variants for implementation, review, and investigation workflows
- define severity tiers for different failure modes

## Long term

- build a lightweight evaluation harness around the markdown tests
- compare behavior before and after rule changes
- support multiple agent styles while preserving the same guardrail standards
