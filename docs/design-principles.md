# Design principles

## 1. Rules must be actionable

Abstract values such as “be careful” or “think harder” are weak. Rules should define concrete behavior.

## 2. Observed failure beats hypothetical advice

Add rules when they correspond to real failure modes, not imagined perfection.

## 3. Verification scales with impact

The stronger the claim, the stronger the evidence requirement.

## 4. Narrow changes are easier to trust

Minimal bug fixes are easier to review, verify, and maintain.

## 5. Ambiguity should be surfaced, not hidden

If scope or intent is unclear, the uncertainty should be made visible instead of silently guessed away.
